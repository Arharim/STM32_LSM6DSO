# Калибровка акселерометра методом 6-ти осей

Этот набор скриптов GNU Octave предназначен для калибровки акселерометра методом 6-ти осей.

## Принцип работы

Метод 6-ти осей основан на том, что акселерометр измеряет силу тяжести (1g) в различных ориентациях:

1. **X вниз** - ось X направлена вниз (должна показывать +1g)
2. **X вверх** - ось X направлена вверх (должна показывать -1g)  
3. **Y вниз** - ось Y направлена вниз (должна показывать +1g)
4. **Y вверх** - ось Y направлена вверх (должна показывать -1g)
5. **Z вниз** - ось Z направлена вниз (должна ��оказывать +1g)
6. **Z вверх** - ось Z направлена вверх (должна показывать -1g)

## Формулы калибровки

Для каждой оси рассчитываются:

### Смещение (Bias)
```
bias_x = (X_down + X_up) / 2
bias_y = (Y_down + Y_up) / 2  
bias_z = (Z_down + Z_up) / 2
```

### Масштабный коэффициент (Scale)
```
scale_x_measured = (X_down - X_up) / 2
scale_x = 1.0 / scale_x_measured

scale_y_measured = (Y_down - Y_up) / 2
scale_y = 1.0 / scale_y_measured

scale_z_measured = (Z_down - Z_up) / 2
scale_z = 1.0 / scale_z_measured
```

### Применение калибровки
```
ax_corrected = (ax_raw - bias_x) * scale_x
ay_corrected = (ay_raw - bias_y) * scale_y
az_corrected = (az_raw - bias_z) * scale_z
```

## Файлы скриптов

### Основные скрипты

1. **`accel_6axis_calibration.m`** - основная функция калибровки
2. **`run_accel_calibration.m`** - скрипт для запуска калибровки
3. **`load_accel_calibration.m`** - загрузка коэффициентов калибровки
4. **`apply_accel_calibration.m`** - применение калибровки к данным
5. **`test_accel_calibration.m`** - тестирование калибровки

### Входные файлы данных

Скрипты ожидают следующие CSV файлы в папке `../logs/`:

- `accel_X_down_calibration_data.csv` - данные с осью X вниз
- `accel_X_up_calibration_data.csv` - данные с осью X вверх
- `accel_Y_down_calibration_data.csv` - данные с осью Y вниз
- `accel_Y_up_calibration_data.csv` - данные с осью Y вверх
- `accel_Z_down_calibration_data.csv` - данные с осью Z вниз
- `accel_Z_up_calibration_data.csv` - данные с осью Z вверх

### Формат CSV файлов

Файлы должны содержать заголовок и данные в формате:
```
Index,gx_dps,gy_dps,gz_dps,ax_g,ay_g,az_g
0,-30.52,+8.54,+8.12,+0.051,-0.018,+1.013
1,-30.66,+7.35,+8.61,+0.066,-0.029,+1.030
...
```

Где:
- Столбцы 4, 5, 6 (ax_g, ay_g, az_g) содержат данные акселерометра в единицах g

## Использование

### 1. Подготовка данных

Соберите данные калибровки для всех 6 позиций и сохраните их в соответствующие CSV файлы.

### 2. Запуск калибровки

```octave
% В GNU Octave
cd plotVis
run_accel_calibration
```

Или напрямую:
```octave
[bias_x, bias_y, bias_z, scale_x, scale_y, scale_z] = accel_6axis_calibration();
```

### 3. Результаты

После выполнения калибровки создаются файлы:

- **`accel_calibration_coefficients.mat`** - коэффициенты для Octave/MATLAB
- **`accel_calibration_coefficients.txt`** - коэффициенты в формате C/C++

### 4. Тестирование

Для проверки качества калибровки:
```octave
test_accel_calibration
```

### 5. Использование в других скриптах

```octave
% Загрузка коэффициентов
[bias_x, bias_y, bias_z, scale_x, scale_y, scale_z] = load_accel_calibration();

% Применение к данным
[ax_cal, ay_cal, az_cal] = apply_accel_calibration(ax_raw, ay_raw, az_raw, ...
                                                   bias_x, bias_y, bias_z, ...
                                                   scale_x, scale_y, scale_z);
```

## Оценка качества калибровки

Скрипт автоматически оценивает качество калибровки по средней абсолютной ошибке:

- **< 0.05g** - Отличное качество
- **< 0.1g** - Хорошее качество  
- **< 0.2g** - Удовлетворительное качество
- **≥ 0.2g** - Плохое качество (требуется повторная калибровка)

## Устранение неполадок

### Ошибка "Файл не найден"
Убедитесь, что все 6 файлов калибровки находятся в папке `../logs/` относительно папки со скриптами.

### Ошибка "Файл пуст"
Проверьте, что CSV файлы содержат данные (не только заголовок).

### Плохое качество калибровки
1. Проверьте правильност�� ориентации датчика при сборе данных
2. Убедитесь, что датчик был неподвижен во время измерений
3. Повторите сбор данных для проблемных позиций

## Интеграция в код микроконтроллера

Используйте коэффициенты из файла `accel_calibration_coefficients.txt`:

```c
#define ACCEL_BIAS_Y  -0.009795f  
#define ACCEL_BIAS_Z  0.029718f

#define ACCEL_SCALE_X 0.993255f
#define ACCEL_SCALE_Y 0.994089f
#define ACCEL_SCALE_Z 1.006982f

// Применение калибровки
float ax_corrected = (ax_raw - ACCEL_BIAS_X) * ACCEL_SCALE_X;
float ay_corrected = (ay_raw - ACCEL_BIAS_Y) * ACCEL_SCALE_Y;
float az_corrected = (az_raw - ACCEL_BIAS_Z) * ACCEL_SCALE_Z;
```
